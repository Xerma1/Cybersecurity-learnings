# What is CVE?
Stands for Common Vulnerabilities and Exposures.<br>

An archive of vulnerabilities that are discovered - a weakness in software or hardware. Each CVE entry has a unique ID to classify each vulnerability.<br>

* CVE-2021-44228: The famous Log4 shell bug

Speeds up communication and response.<br>

But just knowing that the vulnerability exists is not enough. We want to know its severity as well! Say hello to CVSS.

# What is CVSS
Stands for the Common Vulnerability Scoring System. Gives a score between 0-10 for each vulnerability. Higher numbers mean higher severity.<br>

Help organisations prioritise which vulnerabilities to fix.

# What is CWE
Stands for Common Weakness Enumeration. While CVE describes a specific vulnerability, CWE describes a general type of weakness that caused it. 
* Buffer overflow
* Cross-Site Scripting (XSS)
* SQL Injections <br>

Classifies vulnerabilities under similar categories, helping developers understand not just what is vulnerable, but WHY they are vulnerable. <br>

Together, they provide a system that allows the cybersecurity world to organise, prioritise and fix vulnerabilities quickly.
