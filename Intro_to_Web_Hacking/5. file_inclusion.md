# What is file inclusion?
File inclusion is the parameter in the URL when we request a resource from a web server, like this:
<img width="1180" height="563" alt="image" src="https://github.com/user-attachments/assets/6e7bc0fa-f272-40a9-b37b-4a1938a87ea2" />

First, the user sends an HTTP request to the web server that includes a file to display. For example, if a user wants to access and display their CV within the web application, the request may look as follows, `http://webapp.thm/get.php?file=userCV.pdf`, where the `file` is the parameter and the `userCV.pdf` is the required file to access.<br>

File inclusion vulnerabilities occur when the input is not sanitised or validated, leading to the exposure of confidential data like source code, user profile details, and other important files.

# Local File Inclusion (LFI)
For PHP, using functions such as `include`, `require`, `include_once` and `require_once` often contribute to vulnerable web applications.

## LFI scenario 1
```
<?PHP
  include($_GET["lang"]);
?>
```
The above PHP code uses a `GET` request via the URL parameter `lang` to include the file of the page. `http://webapp.thm/index.php?lang=EN.php` to load the English page or `http://webapp.thm/index.php?lang=AR.php` to load the Arabic page.<br>

The user input directly becomes the full file path. If there is no input validation, we can do `htttp://webapp.thm/get.php?file=/etc/passwd`.

## LFI scenario 2
```
<?PHP
  include("languages/". $_GET["lang"]);
?>
```
In the above code, user input is appended with `languages/`, so doing `file=/etc/passwd` will include `languages//etc/passwd`, which is not in the directory. Therefore, we can do `http://webapp.thm/get.php?file=../../../../etc/passwd` to traverse up the directory.
